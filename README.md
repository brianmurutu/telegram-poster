## вңЁ What It Does

This bot monitors your blog's homepage every **15 minutes** using a scheduled GitHub Actions workflow. The moment a new article is detected, it:

- рҹ“ў **Broadcasts** a beautifully formatted post to **multiple Telegram channels** simultaneously
- рҹ“° **Extracts** the article title, meta description, and tags automatically from your blog
- рҹҸ·пёҸ **Converts** your blog's tags into Telegram hashtags dynamically
- рҹ”ҳ **Attaches** inline action buttons (Read Article + Join Channel CTA)
- рҹ”” **Notifies** you personally in your Telegram DMs with a delivery status report
- рҹ’ҫ **Remembers** the last posted article to avoid duplicate posts
- вң… **Fault-tolerant** вҖ” if one channel fails, others still receive the post

**Zero server required. Runs entirely free on GitHub Actions.**

## рҹҡҖ Setup Guide

### Step 1: Create a Telegram Bot

1. Open Telegram and search for **@BotFather**
2. Send the command `/newbot`
3. Follow the prompts вҖ” choose a name and a username for your bot (username must end in `bot`, e.g. `myblogposter_bot`)
4. BotFather will reply with your **Bot Token** вҖ” it looks like:
   ```
   7123456789:AAGcXXXXXXXXXXXXXXXXXXXXXXXXXXX
   ```
5. **Copy and save this token** вҖ” you will need it in Step 6

> вҡ пёҸ **Never share your bot token publicly.** Anyone with it can control your bot.

---

### Step 2: Add Your Bot as Channel Admin

For **each Telegram channel** you want to post to:

1. Open your channel in Telegram
2. Go to **Channel Info вҶ’ Administrators вҶ’ Add Administrator**
3. Search for your bot's username and add it
4. Grant it **Post Messages** permission (minimum required)
5. Save

---

### Step 3: Get Your Personal Chat ID

The bot needs your **numeric user ID** (not your username) to send you personal notifications.

1. Open Telegram and search for **@userinfobot**
2. Send it `/start`
3. It will reply with something like:
   ```
   Id: 123456789
   First: John
   ```
4. **Copy the numeric ID** вҖ” you will use it in the script

> рҹ’Ў Also send `/start` to **your own bot** in a private DM вҖ” this is required before it can message you.

**Changing the check frequency:**

Edit the cron expression in `.github/workflows/telegram-notify.yml`:

```yaml
schedule:
  - cron: '*/15 * * * *'   # Every 15 minutes (default)
  - cron: '*/5 * * * *'    # Every 5 minutes
  - cron: '0 * * * *'      # Every hour
  - cron: '0 9 * * *'      # Once daily at 9am UTC
```

> вҡ пёҸ GitHub Actions free tier has limits. Checking every 5 minutes on an active repo is fine; every 1 minute may hit rate limits.

## рҹӣ  Troubleshooting

**вқҢ Exit code 1 вҖ” KeyError: TELEGRAM_BOT_TOKEN**
Your GitHub secret is missing or misspelled. Go to **Settings вҶ’ Secrets вҶ’ Actions** and confirm the secret is named exactly `TELEGRAM_BOT_TOKEN`.

**вқҢ 403 Forbidden when posting to channel**
Your bot has not been added as an administrator to the channel. See [Step 2](#step-2-add-your-bot-as-channel-admin).

**вқҢ Bot cannot message me personally (admin notification fails)**
Two possible causes:
- You haven't sent `/start` to your bot in a private DM yet
- You used `@username` instead of the numeric user ID. Get your ID from [@userinfobot](https://t.me/userinfobot)

**вқҢ No posts found / script says "Could not find any posts"**
Your blog's HTML structure differs from the default. Open your blog вҶ’ right-click an article title вҶ’ Inspect Element вҶ’ identify the tag wrapping the title link and update the `get_latest_post()` function accordingly.

**вқҢ Exit code 128 вҖ” git push fails**
The workflow couldn't commit `last_post.json` back to the repo. Make sure the `GITHUB_TOKEN` has write permissions: go to **Settings вҶ’ Actions вҶ’ General вҶ’ Workflow permissions** вҶ’ select **Read and write permissions**.

**вҡ пёҸ Posts duplicate on the same channel**
You have the same channel listed twice вҖ” once as a numeric ID and once as a username. They refer to the same channel. Keep only the username format: `@channel_name`.

## вқ“ FAQ

**Does this work with WordPress?**
Yes. WordPress blogs expose article links in `<h2>` tags by default on most themes. If your theme is different, inspect your homepage HTML and update the `get_latest_post()` function to match.

**Does this work with Ghost, Webflow, or custom CMS?**
Yes, as long as the blog homepage lists articles as HTML links. Alternatively, if your platform provides an RSS feed, you can switch back to `feedparser` for a more reliable detection method.

**Can I post to more than two channels?**
Yes. Just add more usernames to the `CHANNELS` list:
```python
CHANNELS = ["@channel_one", "@channel_two", "@channel_three", "@channel_four"]
```

**Will it re-post old articles?**
No. Once an article URL is saved to `last_post.json`, it will never be posted again unless you manually delete that file from the repository.

**Can I add an image/thumbnail to the post?**
Yes вҖ” this requires using Telegram's `sendPhoto` API method instead of `sendMessage`. This is planned for a future version.

**Is this free to run?**
Yes. GitHub Actions provides 2,000 free minutes per month on free accounts. A workflow running every 15 minutes uses roughly 1вҖ“2 minutes per run Г— ~2,880 runs/month = well within free tier limits.

**What if GitHub Actions is down?**
Posts will be delayed until the next successful workflow run. The bot will catch up and post when Actions resumes, as long as the homepage still shows the new article.
